# Problema

O sistema resolve a incerteza operacional de clientes que precisam usar M-Pesa presencialmente e nao sabem se o agente esta disponivel, presente ou capaz de atender o pedido.

No comportamento actual, essa incerteza e reduzida por:

- mapa com agentes e estado operacional;
- pedido de disponibilidade/liquidez antes da deslocacao;
- notificacoes em tempo real;
- reserva temporaria com token;
- dashboard para o agente organizar pedidos;
- metricas para admin acompanhar utilizadores, agentes, pings e zonas criticas.

O sistema nao confirma saldo real do agente nem se integra com a rede M-Pesa. A disponibilidade e operacional, baseada na resposta do agente dentro do sistema.
